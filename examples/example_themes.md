# Example Thematic Analysis Results

This document shows example themes identified through thematic analysis of the qualitative datasets. These examples demonstrate how to code and synthesize qualitative data into actionable insights.

## Example 1: Mobile App Usability Interviews

**Dataset:** `user_interviews/mobile_app_usability_interviews.txt`

**Method:** Inductive Thematic Analysis

### Theme 1: Navigation Confusion

**Description:** Users struggle to find features and navigate the app. Navigation is not intuitive, and features are buried or hidden.

**Codes:**
- Can't find features
- Buried navigation
- Unclear menu structure
- Hidden options
- Confusing labels

**Example Quotes:**
- "I'm looking for... where do I send money? Okay, I see a 'Pay' button at the bottom. Let me tap that. Hmm, it's asking me to add a recipient first."
- "I don't see a clear 'Transactions' or 'History' button. Maybe it's in the account details?"

**Impact:** High - affects core user tasks
**Frequency:** 8 out of 10 participants mentioned this

**Recommendations:**
- Simplify navigation structure
- Make primary actions more prominent
- Improve labeling and discoverability
- Add breadcrumbs or progress indicators

### Theme 2: Information Architecture Issues

**Description:** Information is not organized in a way that matches user mental models. Related information is separated, and hierarchy is unclear.

**Codes:**
- Unclear information hierarchy
- Mismatched mental models
- Separated related information
- Missing context

**Example Quotes:**
- "I can see a number on the screen, but I'm not sure if that's right. It seems low. Let me check... oh, I need to refresh?"
- "The balance is right there, which is good. But... is this my checking or savings? It just says 'Balance: $2,450'. Which account is that?"

**Impact:** High - affects user confidence and decision-making
**Frequency:** 7 out of 10 participants mentioned this

**Recommendations:**
- Show account context prominently
- Group related information together
- Improve visual hierarchy
- Add account type labels

### Theme 3: Transaction History Accessibility

**Description:** Transaction history is difficult to access and lacks useful features like search, filtering, and date grouping.

**Codes:**
- Hard to find transaction history
- No search functionality
- No filtering options
- Poor date organization
- Can't see older transactions

**Example Quotes:**
- "I'm looking for my recent transactions. Let me check... I don't see a clear 'Transactions' or 'History' button."
- "I can see recent transactions, but I can't scroll back very far. And there's no search. So if I'm looking for something from a month ago, I'm out of luck."

**Impact:** Medium-High - affects frequent use case
**Frequency:** 9 out of 10 participants mentioned this

**Recommendations:**
- Make transaction history more prominent
- Add search functionality
- Add date filters and grouping
- Improve scrolling/pagination

### Theme 4: Security vs. Usability Tension

**Description:** Security measures create friction in the user experience. Users want security but find current measures excessive or confusing.

**Codes:**
- Too many security steps
- Confusing verification
- Security creates friction
- Want security but simpler
- Unclear security purpose

**Example Quotes:**
- "I like that there are security checks, but they're too frequent. I'm transferring $20 to a friend, and it's asking me to verify my identity three times. That's overkill."
- "I'm worried I'll send it to the wrong person. The confirmation screen doesn't show enough information for me to be confident I'm doing it right."

**Impact:** Medium - affects user satisfaction
**Frequency:** 6 out of 10 participants mentioned this

**Recommendations:**
- Balance security with usability
- Use risk-based authentication
- Improve confirmation screens
- Explain security measures clearly

### Theme 5: Mobile-Specific Challenges

**Description:** Mobile experience has unique challenges including small screens, keyboard interference, and touch target sizes.

**Codes:**
- Small screen issues
- Keyboard covers fields
- Touch targets too small
- Hard to read on mobile
- Mobile vs. desktop differences

**Example Quotes:**
- "I was trying to checkout on my phone, but the form was too small and hard to fill out. The keyboard kept covering up the fields I was trying to fill."
- "The numbers are small - I have to squint to read them. Can we make the font bigger?"

**Impact:** Medium - affects mobile users specifically
**Frequency:** 5 out of 10 participants mentioned this

**Recommendations:**
- Optimize forms for mobile
- Increase touch target sizes
- Improve font sizes and readability
- Handle keyboard interactions better

## Example 2: Remote Work Diary Study

**Dataset:** `diary_studies/remote_work_diary_entries.csv`

**Method:** Deductive Thematic Analysis (based on work-life balance framework)

### Theme 1: Boundary Blurring

**Description:** The line between work and home life is increasingly blurred. Work extends into personal time, and home distractions affect work.

**Codes:**
- No clear work/home boundary
- Working after hours
- Checking work outside hours
- Home distractions during work
- Difficulty disconnecting

**Example Quotes:**
- "I'm realizing I'm working longer hours from home than I did in the office. There's no clear boundary between work and home anymore."
- "I'm checking Slack at 8pm, answering emails on weekends. I need to set better boundaries, but it's hard when everyone else is always online."

**Impact:** High - affects work-life balance and well-being
**Frequency:** 8 out of 10 participants mentioned this

**Recommendations:**
- Set clear work hours
- Use "do not disturb" features
- Create physical separation
- Establish team norms

### Theme 2: Technology Challenges

**Description:** Technical issues including internet connectivity, equipment problems, and software issues create frustration and reduce productivity.

**Codes:**
- Internet connectivity issues
- Equipment problems
- Software crashes
- Slow performance
- Technology barriers

**Example Quotes:**
- "My internet keeps cutting out during important calls. My laptop is slow and crashes sometimes. I don't have a proper monitor, so I'm squinting at a small screen all day."
- "Technology issues slowed me down all day. Had to restart my computer three times. Lost work because something didn't save properly."

**Impact:** High - directly affects productivity
**Frequency:** 7 out of 10 participants mentioned this

**Recommendations:**
- Provide equipment support
- Improve IT infrastructure
- Offer technical assistance
- Backup and recovery systems

### Theme 3: Social Isolation

**Description:** Lack of in-person interaction leads to feelings of loneliness and disconnection from colleagues.

**Codes:**
- Missing social interaction
- Feeling isolated
- Loneliness
- Lack of casual conversation
- Video calls not the same

**Example Quotes:**
- "Loneliness. I miss seeing people. Video calls aren't the same. I miss casual conversations, coffee breaks, just being around other humans."
- "I'm an extrovert, and this is really hard for me. I've been going to co-working spaces sometimes, but it's expensive and not the same as being with my team."

**Impact:** Medium-High - affects mental health and job satisfaction
**Frequency:** 6 out of 10 participants mentioned this

**Recommendations:**
- Schedule regular team check-ins
- Create virtual social spaces
- Encourage informal communication
- Support co-working options

### Theme 4: Meeting Overload

**Description:** Excessive meetings consume time and energy, leaving little time for actual work.

**Codes:**
- Too many meetings
- Back-to-back calls
- No time for work
- Meeting fatigue
- Unnecessary meetings

**Example Quotes:**
- "Video calls. I've had 5 calls today already, and it's only 2:30pm. Each one could have been an email or Slack message."
- "I have 6 hours of meetings scheduled today, which leaves no time for actual work. I'm going to have to work tonight to catch up."

**Impact:** High - affects productivity and work-life balance
**Frequency:** 9 out of 10 participants mentioned this

**Recommendations:**
- Reduce meeting frequency
- Make meetings async when possible
- Set meeting-free days
- Improve meeting efficiency

### Theme 5: Productivity Variations

**Description:** Productivity varies significantly based on individual circumstances, time of day, and environmental factors.

**Codes:**
- Varying productivity levels
- Time of day effects
- Environmental factors
- Individual differences
- Finding optimal conditions

**Example Quotes:**
- "Up early, which I prefer. Quiet house, no distractions. Made coffee, sat down, and got straight to work. This is when I'm most productive."
- "Got through it, but barely. Technology issues slowed me down all day. Had to restart my computer three times."

**Impact:** Medium - affects individual performance
**Frequency:** All participants showed variation

**Recommendations:**
- Allow flexible schedules
- Support individual preferences
- Provide quiet work spaces
- Recognize individual differences

## Example 3: Healthcare Portal Interviews

**Dataset:** `user_interviews/healthcare_portal_interviews.json`

**Method:** Hybrid Thematic Analysis (inductive + deductive)

### Theme 1: Technology Anxiety

**Description:** Users, especially those with low tech comfort, feel anxious and overwhelmed by technology requirements.

**Codes:**
- Fear of technology
- Worry about mistakes
- Overwhelmed by complexity
- Lack of confidence
- Prefer human interaction

**Example Quotes:**
- "I'm not very good with computers, so I'm worried about this. My doctor said I need to use this portal to see my test results, but I don't know how."
- "I'm worried I'll tap the wrong thing and send money to the wrong person or something. I need more confidence that I'm doing things right."

**Impact:** High - prevents adoption and use
**Frequency:** 4 out of 10 participants (all low tech comfort)

**Recommendations:**
- Provide clear guidance and help
- Offer alternative access methods
- Design for low tech comfort users
- Build confidence through success

### Theme 2: Information Clarity

**Description:** Medical information is presented without sufficient explanation or context, making it difficult to understand.

**Codes:**
- Unclear medical terminology
- Lack of explanation
- No context for results
- Hard to understand
- Need interpretation

**Example Quotes:**
- "I can see that I have test results, but I can't read them. The text is too small, and there are a lot of numbers and abbreviations that I don't understand."
- "I can find my test results, but I can't understand them. The medical terminology isn't explained, and there's no context about what's normal."

**Impact:** High - affects health outcomes
**Frequency:** 7 out of 10 participants mentioned this

**Recommendations:**
- Explain medical terminology
- Provide context and normal ranges
- Use plain language
- Offer interpretation support

### Theme 3: Feature Discoverability

**Description:** Features are difficult to find and use, especially messaging and test results.

**Codes:**
- Hard to find features
- Buried functionality
- Unclear how to use
- Missing guidance
- Poor discoverability

**Example Quotes:**
- "I wanted to ask my doctor a question, but I couldn't figure out how to send a message. I saw something that said 'Messages' but when I clicked it, I didn't see how to write a new message."
- "I couldn't find my test results anywhere. I clicked around for maybe 20 minutes, and I still couldn't find them."

**Impact:** High - affects core functionality
**Frequency:** 8 out of 10 participants mentioned this

**Recommendations:**
- Improve navigation and labeling
- Add contextual help
- Make primary actions prominent
- Provide clear guidance

## Key Insights from Thematic Analysis

1. **Navigation and discoverability** are common issues across multiple datasets
2. **Technology anxiety** affects adoption, especially for vulnerable populations
3. **Information clarity** is critical for health and financial contexts
4. **Boundary management** is a challenge in remote work
5. **Security-usability balance** needs careful consideration

## Using Thematic Analysis in UX Research

**Benefits:**
- Deep understanding of user experiences
- Identifies patterns and relationships
- Provides rich, contextual insights
- Supports design decisions

**Process:**
1. Familiarize with data
2. Generate initial codes
3. Search for themes
4. Review themes
5. Define and name themes
6. Write up analysis

**Best Practices:**
- Use both inductive and deductive approaches
- Validate themes with data
- Consider negative cases
- Document your process
- Get peer review
